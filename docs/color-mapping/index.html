Q. 線形回帰は好きですか？

線形回帰

  • 値の組を突っ込むと別の値がいい感じに出てくるような線形結合の表現を求めるやつ
  • 観測可能な値の組から未知の別の値のほうを予測したりするのに使う
  • 詳しいことは各自勉強してください

しかし、線形回帰には致命的な弱点が😨

ふつうに生活してて、線形回帰に突っ込みたくなるようなデータとの出会いなんかないですよ……

そんな線形回帰を救いたい

#' Decode nativeRaster into a tibble
#' @param nr A nativeRaster object.
#' @param to Passed to `to` argument of [farver::decode_colour()].
#' @returns A tibble.
decode_icol <- function(nr, to = "rgb") {
  if (!inherits(nr, "nativeRaster")) {
    cli::cli_abort("Input must be a nativeRaster object")
  }
  rgb <-
    farver::decode_native(nr) |>
    farver::decode_colour(alpha = FALSE, to = to)

  dplyr::tibble(
    icol = as.integer(nr),
    r = rgb[, 1],
    g = rgb[, 2],
    b = rgb[, 3],
    a = if (to == "cmyk") rgb[, 4] else 1
  )
}

suppressPackageStartupMessages({
  library(tidymodels)
})

color_split <- initial_split(dat, prop = 0.1, strata = icol)
color_train <- training(color_split)

color_rec <-
  recipe(icol ~ r + g + b, data = color_train) |>
  # そのまま回帰すると完全に予測できてしまうので、非線形な変換をかける
  step_cut(all_numeric_predictors(), breaks = 100)

color_spec_lm <-
  linear_reg() |>
  set_engine("lm")

color_wf <-
  workflow() |>
  add_recipe(color_rec) |>
  add_model(color_spec_lm) |>
  fit(color_train)

pred <-
  augment(
    extract_fit_parsnip(color_wf),
    new_data = prep(color_rec) |> bake(new_data = dat)
  ) |>
  dplyr::select(.pred, icol)

# metricsの確認。そこそこ外れているほうがおもしろい
pred |>
  metrics(truth = icol, estimate = .pred)

as.nativeRaster <- function(x, size, ...) {
  UseMethod("as.nativeRaster")
}

as.nativeRaster.numeric <- function(x, size, ...) {
  if (length(size) != 2 || length(x) != prod(size)) {
    cli::cli_abort("Input must be of length {prod(size)}")
  }
  x <- structure(as.integer(x), class = "nativeRaster")
  dim(x) <- size
  x
}

out_dir <- tempdir()

imgs <- purrr::imap_chr(fs::dir_ls("colormap"), purrr::in_parallel(\(f, i) {
  library(tidymodels)

  pict <- aznyan::read_still(f)
  rgb <-
    farver::decode_native(pict) |>
    farver::decode_colour(to = "rgb")
  dat <-
    dplyr::tibble(
      icol = as.integer(pict),
      r = rgb[, 1],
      g = rgb[, 2],
      b = rgb[, 3]

  pred <-
    augment(
      model,
      new_data = prep(recipe) |> bake(new_data = dat)
    ) |>
    dplyr::select(.pred, icol)

  predicted_img <- as.integer(pred$.pred)
  dim(predicted_img) <- dim(pict)
  class(predicted_img) <- "nativeRaster"

  fp <- file.path(out_dir, basename(f))
  aznyan::write_still(predicted_img, fp)

まとめ

  • 画像はテーブルデータと見なせば回帰分析に突っ込める
  • わざとunderfitさせた回帰モデルを使って別の画像の色を予測することによって、画像の色を変換できるぞ！
  • カラーフィルターに転用することによって救われる線形回帰があるかもしれない
